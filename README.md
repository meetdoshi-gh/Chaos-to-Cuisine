# Chaos to Cuisine — QMIX Cooperative MARL System

A cooperative multi-agent reinforcement learning system that trains two independent agents to solve a shared sequential task under sparse rewards, using value decomposition with a monotonic mixing network to guarantee globally consistent decentralized execution.

## System Overview

The system implements a Centralized Training with Decentralized Execution (CTDE) architecture. During training, a centralized mixing component has access to the full joint state and computes a team value function from individual agent utilities. During deployment, each agent acts independently using only its own observation. This asymmetry is the mechanism that enables both effective credit assignment during learning and scalable autonomous execution at test time.

The key algorithmic contribution is ensuring the Individual Global Max (IGM) property holds by construction: the joint action maximizing the team value is always the composition of individually optimal actions, guaranteed by monotonicity constraints baked into the mixing network's architecture.

## Key Algorithms Explained

**Value Decomposition:** Instead of learning one large joint action-value function over all possible agent combinations, the team value Q_tot is factored into per-agent utility functions Q_i, combined through a learned mixing function. Each Q_i depends only on the individual agent's observation and action, making decentralized execution tractable.

**Monotonic Mixing:** The mixing network accepts the individual utility values and produces Q_tot via a two-layer feed-forward computation. All mixing weights are generated by hypernetworks conditioned on the joint state, with a non-negativity constraint on the weights ensuring the monotonicity condition required for IGM.

**Off-Policy Learning:** Agent networks and the mixing network are trained jointly on mini-batches sampled from a replay buffer of past transitions. Separate target networks, updated through Polyak averaging, provide stable regression targets for the Bellman TD loss.

## Architecture Description

The system consists of three categories of neural components:

**Agent Networks:** Each agent maintains an online DQN and a target DQN. The online network maps the agent's observation to Q-values for each possible action. The target network provides stable Bellman targets and is periodically soft-updated toward the online network.

**Mixing Network:** A two-layer feed-forward network that takes the tuple of agent Q-values (for actions taken) and produces a scalar Q_tot. The network has no fixed weights — all parameters are generated dynamically by the hypernetworks as functions of the current joint state.

**Hypernetworks:** Four feed-forward networks that consume the joint state and output the weight matrices and bias vectors for the mixing network's two layers. Weight-generating hypernetworks use a non-negative activation function; bias-generating hypernetworks are unconstrained.

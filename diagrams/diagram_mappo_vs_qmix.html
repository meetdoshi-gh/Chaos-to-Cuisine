Why QMIX? — Algorithm Selection for Cooperative MARL
Context: Three MARL algorithms were evaluated — MAPPO, MADDPG, and QMIX. The choice is structural, not stylistic: each algorithm makes fundamentally different assumptions about the action space, training paradigm, and credit assignment mechanism. For the Overcooked setting (discrete actions, fully cooperative, sparse rewards, slow simulator), these differences determine success or failure.
Considered — not chosen
MAPPO
Multi-Agent PPO · On-Policy Actor-Critic
Training Paradigm
On-Policy
Cannot reuse experience. Every update requires fresh rollouts — expensive with a slow simulator. No replay buffer compatibility.
Team Value Function
Centralized V(s)
Shared state-value critic estimates return but produces no per-agent decomposition. No value factorization — credit assigned only through GAE statistics.
Credit Assignment
Indirect via GAE
Generalized Advantage Estimation approximates per-agent credit from V(s). Noisy and indirect — no explicit per-agent utility signal produced.
IGM Property
Not Guaranteed
No monotonicity constraint. Local greedy actions are not provably consistent with the global optimum — coordination is statistically encouraged, not architecturally enforced.
Hyperparameter Surface
Broader — 3 extra knobs
Adds: clip range ε, GAE λ, entropy coefficient — each requiring separate tuning beyond standard RL hyperparameters.
Action Space Fit
Discrete ✓
Categorical policy over discrete actions — compatible with Overcooked's 6-action space. This is not a disadvantage vs QMIX.
Chosen ✓
QMIX
Monotonic Value Decomposition · Off-Policy Q-Learning
Training Paradigm
Off-Policy + Replay
Decouples data collection from optimization. Replay buffer stores 100K transitions — each reused many times. Critical for sample efficiency in slow simulators.
Team Value · CORE
Q_tot via Decomposition
Q_tot explicitly factored into per-agent utilities Q_i via hypernetwork mixing. Direct, architecturally grounded — not approximated through gradient statistics.
Credit Assignment
Explicit via Mixing ∇
TD loss gradients flow through mixing weights back to each agent. Each agent receives credit proportional to its contribution to Q_tot — explicit and architecturally grounded.
IGM Property
Architecturally Guaranteed
Softplus on mixing weights enforces ∂Q_tot/∂Qᵢ > 0 everywhere. Monotonicity is built into the network structure — not trained, not encouraged, guaranteed.
Hyperparameter Surface
Lean — no extra knobs
No clip range, no GAE λ, no entropy coefficient. Core knobs: γ, α, τ, ε-decay — all already necessary for standard Q-learning. Zero additional complexity.
Known Tradeoff ⚡
ε-Decay Is a Razor
Off-policy externalize exploration — ε-decay is razor's edge. In cooperative sparse-reward tasks, one λ value works. All neighbors fail. This is the project's central challenge.
